# PRD 功能审查与 MVP 对齐

**目标**: 将 PRD 中的宏大愿景与 MVP 现实对齐  
**原则**: 保留核心价值，延迟复杂功能

| 功能 | 优先级 | MVP 状态 |
|------|--------|---------|

## 关键修正总结

### 1. 不做 MCP

**之前错误**: 规划了 MCP 整合  
**PRD 明确**: "MCP 会导致 Context 爆炸，违背轻量理念"  
**修正**: 从所有任务中移除 MCP

### 2. 简化记忆

**PRD 设计**: 三层记忆（短/工/长）  
**MVP 现实**: 单层 Session 足够  
**修正**: 简化实现，保留概念

### 3. 延迟学习系统

**PRD 愿景**: 自适应学习，越用越好  
**MVP 现实**: 复杂且难以验证  
**修正**: 用固定 Skill 替代，学习系统延后
