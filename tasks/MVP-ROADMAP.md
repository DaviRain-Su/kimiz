# Kimiz MVP 路线图

**目标**: 从 18k 行"实验性功能"转向稳定的 MVP 产品  
**核心策略**: 先做好一件事（可靠的 REPL + 文件编辑）  
**时间线**: 6 周迭代

---

## MVP 定义

### 什么是最小可用产品？

一个**可靠的命令行 AI 助手**，能够：

1. **启动 REPL** → 与 AI 对话
2. **读取文件** → 了解项目
3. **编辑文件** → 修改代码
4. **记住上下文** → 同一会话内保持连贯

**仅此而已。**

### MVP 范围

| 功能 | 包含 | 排除 |
|------|------|------|
| 对话 | ✅ REPL 模式 | ❌ Web UI |
| 文件操作 | ✅ read/edit/write | ❌ 复杂重构 |
| AI 模型 | ✅ **Kimi (kimi-k2.5)** | ❌ 多 Provider |
| 记忆 | ✅ Session 级 | ❌ 跨会话 / 三层记忆 |
| 工具 | ✅ 5 个核心工具 | ❌ MCP / 外部工具 |
| 流程 | ✅ 简单 Agent 循环 | ❌ Harness / SubAgent |
| 学习 | ❌ 无 | ❌ 用户偏好学习 |
