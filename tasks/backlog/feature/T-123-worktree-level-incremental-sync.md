### T-123: worktree-level-incremental-sync
**状态**: pending
**优先级**: P0
**创建**: 2026-04-05
**预计耗时**: 20h

**描述**:
实现 Worktree 级别的增量同步机制，让子代理能够选择性合并其他 worktree 的已完成变更。

在物理隔离和协同工作之间找到平衡。需要：
1. 设计变更检测机制：扫描其他 worktree 中已通过测试/验证的改动
2. 实现可选的 cherry-pick / merge 工作流，将特定改动同步到当前 worktree
3. 在同步前进行冲突预判，如果存在冲突则通过事件总线通知相关 agent
4. 支持自动同步策略配置（如：自动合并同文件的非冲突修改）
5. 确保同步后的代码仍然能通过编译和测试
