## Migration Patterns (Reference Implementation)

### Pattern 1: Read File (Most Common)
```zig
// OLD
const file = std.fs.cwd().openFile(path, .{}) catch |err| { ... };
defer file.close();
const content = try file.readToEndAlloc(allocator, max_size);

// NEW
const content = utils.readFileAlloc(allocator, path, max_size) catch |err| { ... };
defer allocator.free(content);
```

### Pattern 2: Write File
```zig
// OLD
const file = try std.fs.cwd().createFile(path, .{});
defer file.close();
try file.writeAll(content);

// NEW
try utils.writeFile(path, content);
```

### Pattern 3: Directory Iteration
```zig
// OLD
const dir = try std.fs.cwd().openDir(path, .{ .iterate = true });
defer dir.close();
var it = dir.iterate();
while (try it.next()) |entry| { ... }

// NEW
const io = try utils.getIo();
const dir = try utils.openDir(path, .{ .iterate = true });
defer dir.close(io);
var it = dir.iterate();
while (try it.next(io)) |entry| { ... }
```

### Pattern 4: Create Directory
```zig
// OLD
std.fs.cwd().makeDir(path) catch |e| switch (e) {
    error.PathAlreadyExists => {},
    else => return e,
};

// NEW
utils.makeDirRecursive(path) catch |e| switch (e) {
    error.PathAlreadyExists => {},
    else => return e,
};
```

### Pattern 5: File Existence Check
```zig
// OLD
std.fs.cwd().access(path, .{}) == {}

// NEW
utils.fileExists(path)
```

## Risk Mitigation

### High Risk Areas
1. **File Handle管理** - 某些代码可能依赖File句柄的生命周期
   - 解决：仔细检查defer和error handling
2. **Error Handling变化** - utils函数可能有不同的错误类型
   - 解决：保持原有错误处理逻辑
3. **Performance** - utils包装可能引入开销
   - 解决：仅用于Zig 0.16兼容，未来可优化

### Rollback Plan
- Git commit分阶段提交
- 每个模块独立commit
- 出现问题可回滚单个模块
