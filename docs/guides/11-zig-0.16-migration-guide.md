# Zig 0.16 迁移指南

**文档日期**: 2026-04-05  
**当前版本**: Zig 0.15.2  
**目标版本**: Zig 0.16.0

---

## 概述

Zig 0.16 引入了**重大的破坏性变更**，主要集中在：
1. 🔴 **std.Io 接口重构** - 异步 I/O 系统大改
2. 🔴 **std.fs/std.net API 迁移**
3. 🔴 **Build 系统变更**
4. 🔴 **main() 函数签名变化**
5. 🟡 **ArrayList API 变化**

---

## 1. std.Io 接口重构 🔴 **重大变更**

### 1.1 新的 I/O 模型

Zig 0.16 引入了全新的 `std.Io` 接口，支持：
- **io_uring** (Linux)
- **Grand Central Dispatch** (macOS)
- **线程池** (fallback)

**核心概念**:
```zig
// 0.16 新增
std.Io          // I/O 接口抽象
std.Io.Threaded // 基于线程池的实现
std.Io.Evented  // 基于事件驱动 (io_uring/GCD)
```

### 1.2 I/O 对象获取

**所有 I/O 操作都需要一个 `std.Io` 实例**：

```zig
// 方式1: 从 build.zig
const io = b.graph.io; // std.Build 中

// 方式2: 从 main 参数
pub fn main(init: std.process.Init) !void {
    const io = init.io; // ✅ 0.16 新增
}

// 方式3: 自己创建 Threaded
var threaded: std.Io.Threaded = .init(allocator);
defer threaded.deinit();
const io = threaded.io();

// 方式4: 自己创建 Evented
var evented: std.Io.Evented = .init(allocator);
defer evented.deinit();
const io = evented.io();
```

## 2. std.fs → std.Io.Dir/File 🔴

### 2.1 API 迁移

| Zig 0.15.2 | Zig 0.16 | 说明 |
|------------|----------|------|
| `std.fs.Dir` | `std.Io.Dir` | 目录类型 |
| `std.fs.File` | `std.Io.File` | 文件类型 |
| `std.fs.cwd()` | `std.Io.Dir.cwd()` | 获取当前目录 |
| `std.fs.openFileAbsolute()` | `std.Io.File.open()` | 打开文件 |

### 2.2 函数签名变化

**第一个参数现在是 `io: *std.Io`**：

```zig
// Zig 0.15.2
const file = try std.fs.cwd().openFile("config.json", .{});

// Zig 0.16
const io = // ... 从某处获取
const dir = std.Io.Dir.cwd();
const file = try dir.openFile(io, "config.json", .{});
```

**影响的文件**:
- `src/http.zig` - 文件 I/O
- `src/utils/config.zig` - 配置文件读写
- `src/utils/session.zig` - 会话文件
- `src/utils/log.zig` - 日志文件

## 3. std.net → std.Io.net 🔴

### 3.1 API 迁移

| Zig 0.15.2 | Zig 0.16 | 说明 |
|------------|----------|------|
| `std.net.getAddressList()` | `std.Io.net.HostName.lookup()` | DNS 查询 |
| `std.net.Address` | `std.Io.net.Address` | 网络地址 |
| `std.net.tcpConnectToAddress()` | `std.Io.net.TcpSocket.connect()` | TCP 连接 |

### 3.2 DNS 查询重写

```zig
// Zig 0.15.2
const result = try std.net.getAddressList(allocator, "example.com", 80);
defer result.deinit();
for (result.addrs) |addr| {
    // ...
}

// Zig 0.16
const hostname: std.Io.net.HostName = try .init("example.com");
var elem_buf: [16]std.Io.net.HostName.LookupResult = undefined;
var queue: std.Io.Queue(std.Io.net.HostName.LookupResult) = .init(&elem_buf);
var cname_buf: [std.Io.net.HostName.max_len]u8 = undefined;

std.Io.net.HostName.lookup(hostname, io, &queue, .{
    .port = 80,
    .canonical_name_buffer = &cname_buf,
});

while (queue.getOne(io)) |result| {
    switch (result) {
        .address => |addr| {
            // ...
        },
        .canonical_name => |name| {
            // ...
        },
    }
}
```

**影响的文件**:
- `src/http.zig` - HTTP 客户端网络请求

## 4. Build 系统变更 🔴

### 4.1 linkLibC 移动

```zig
// Zig 0.15.2
exe.linkLibC();

// Zig 0.16
const mod = b.addModule("MOD_NAME", .{
    .link_libc = true, // ✅ 移到 Module
});
```

### 4.2 C 源文件添加

```zig
// Zig 0.15.2
exe.addCSourceFiles(.{
    .files = &.{"src/foo.c"},
});

// Zig 0.16
exe.root_module.addCSourceFiles(.{ // ✅ 移到 root_module
    .files = &.{"src/foo.c"},
});
```

### 4.3 linkSystemLibrary 参数变化

```zig
// Zig 0.15.2
mod.linkSystemLibrary("sqlite3");

// Zig 0.16
mod.linkSystemLibrary("sqlite3", .{}); // ✅ 需要空 options
```

### 4.4 captureStdOut 参数变化

```zig
// Zig 0.15.2
run.captureStdOut();

// Zig 0.16
run.captureStdOut(.{}); // ✅ 需要空 options
```

**影响的文件**:
- `build.zig`

## 5. main() 函数签名变化 🔴

### 5.1 新的参数

```zig
// Zig 0.15.2
pub fn main() !void {
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();
    
    const args = try std.process.argsAlloc(allocator);
    defer std.process.argsFree(allocator, args);
}

// Zig 0.16
pub fn main(init: std.process.Init) !u8 {
    // ✅ allocator 已提供
    const allocator = init.gpa;
    
    // ✅ IO 已提供
    const io = init.io;
    
    // ✅ 环境变量已提供
    const env_map = init.environ_map;
    
    // ✅ 参数已提供
    var argv = try init.minimal.args.toSlice(init.gpa);
    defer init.gpa.free(argv);
    
    // ✅ 返回 exit code
    return 0;
}
```

### 5.2 std.process.Init 结构

```zig
pub const Init = struct {
    /// 最小化参数
    minimal: Minimal,
    
    /// 进程级 arena allocator (自动清理)
    arena: *std.heap.ArenaAllocator,
    
    /// 通用 allocator (GPA)
    gpa: Allocator,
    
    /// I/O 实例 (Threaded 或 Evented)
    io: Io,
    
    /// 环境变量 map
    environ_map: *Environ.Map,
    
    /// 预打开的文件句柄 (WASI)
    preopens: Preopens,
    
    pub const Minimal = struct {
        /// 环境变量 (只读)
        environ: Environ,
        /// 命令行参数 (只读)
        args: Args,
    };
};
```

### 5.3 替代方案

如果不想修改 main 签名，可以使用 `Minimal`:

```zig
pub fn main(minimal: std.process.Init.Minimal) !void {
    // 只能访问环境变量和参数
    // 需要自己创建 allocator 和 io
}
```

**影响的文件**:
- `src/main.zig`
- `src/cli/root.zig`

## 6. ArrayList API 变化 🟡

### 6.1 init() → .empty

```zig
// Zig 0.15.2
var list = std.ArrayList(u8).init(allocator);

// Zig 0.16
var list: std.ArrayList(u8) = .empty; // ✅ 不需要 allocator
// 或
var list: std.ArrayList(u8) = .initBuffer(&existing_buffer);
```

### 6.2 writer() 不再接受 allocator

```zig
// Zig 0.15.2
var body_list = std.ArrayList(u8).init(allocator);
const writer = body_list.writer(allocator); // ❌ 0.16 编译错误

// Zig 0.16
var body_list: std.ArrayList(u8) = .empty;
const writer = body_list.writer(); // ✅ 无需 allocator
```

**这正是当前项目的 2 个编译错误之一！**

**影响的文件**:
- `src/http.zig:91` - ❌ 当前编译错误

// Zig 0.15.2
const argv = std.os.argv;

## 8. stdout/stdin API 变化 🔴

### 8.1 获取标准输入输出

```zig
// Zig 0.15.2
const stdout = std.io.getStdOut().writer();
const stdin = std.io.getStdIn().reader();

// Zig 0.16 - 方式1: 使用 std.io (不变)
const stdout = std.io.getStdOut().writer(); // ✅ 仍可用
const stdin = std.io.getStdIn().reader();   // ✅ 仍可用

// Zig 0.16 - 方式2: 使用 Init.io
pub fn main(init: std.process.Init) !u8 {
    const stdout = init.io.stdout.writer();
    const stdin = init.io.stdin.reader();
}
```

**注意**: 当前项目 CLI 中使用的 `stdout().writer(&buffer)` 是错误的 API！

**影响的文件**:
- `src/cli/root.zig:128` - ❌ 当前编译错误 (多处)

## 11. 兼容性策略

### 选项A: 条件编译 (推荐)

```zig
const builtin = @import("builtin");

fn openConfigFile(io: anytype) !std.Io.File {
    if (@import("builtin").zig_version.minor >= 16) {
        // Zig 0.16+
        const dir = std.Io.Dir.cwd();
        return try dir.openFile(io, "config.json", .{});
    } else {
        // Zig 0.15
        return try std.fs.cwd().openFile("config.json", .{});
    }
}
```

### 选项B: 完全迁移到 0.16

- ✅ 代码更简洁
- ✅ 利用新特性 (io_uring, GCD)
- ❌ 不兼容 0.15.2
- ❌ 需要用户升级 Zig

### 选项C: 保持 0.15.2

- ✅ 稳定
- ❌ 无法使用新特性
- ❌ 未来迁移成本更高

**建议**: 选择 **选项A (条件编译)** 或 **选项B (完全迁移)**

## 14. FAQ

**Q: 必须立即升级到 0.16 吗？**
A: 不必须。但 0.16 带来显著性能提升 (io_uring)，建议在方便时迁移。

**Q: 可以继续使用 std.fs 吗？**
A: 不可以。0.16 中 std.fs 已完全移除，必须使用 std.Io.Dir/File。

**Q: main() 函数必须改签名吗？**
A: 是的。虽然可以用 Minimal，但完整的 Init 提供了更多便利。

**Q: 如何获取 io 实例？**
A: 三种方式：1) main 参数 init.io  2) build 中的 b.graph.io  3) 自己创建 Threaded/Evented

**Q: 性能会提升吗？**
A: 是的。在 Linux 上使用 io_uring 可显著提升 I/O 性能，macOS 上使用 GCD。
